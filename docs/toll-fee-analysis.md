## 分析方法

### 第一步：提取所有通行费文件的行程信息

使用 `pdftotext` 提取 PDF 内容，查找关键词：
- "入口"、"出口"：判断行程方向
- "金额"、"发票号码"：确认金额和唯一性
- "时间"：确认日期

### 第二步：根据入口/出口判断行程方向

```
入口：深中侧接线 - 广东机场站沿江，出口：广珠东线 - 广东珠海站
→ 深圳机场 → 珠海

入口：广深高速 - 广东宝安站，出口：中开高速 - 广东翠亨东站
→ 深圳 → 珠海（返程方向）

入口：珠海，出口：广东机场接线
→ 珠海 → 深圳机场
```

### 第三步：匹配通行费与打车行程

- 通行费日期应与打车日期一致或接近
- 通行费金额应与滴滴行程里程合理对应（约 1 元/公里）
- 司机空车返程通行费通常与载客行程通行费金额接近

### 第四步：金额核对

```
滴滴发票金额 + 通行费金额 ≈ 实际付款金额
```

- 差额 < 10 元：可能是平台服务费，可接受
- 差额 > 10 元：检查是否遗漏通行费或其他费用

## 关键要点

1. **通行费放入打车费文件夹**：不要单独建立通行费文件夹
2. **区分去程/返程**：根据入口/出口信息判断，不要假设
3. **包含司机空车返程**：跨市行程通常包含司机返程通行费
4. **文件名唯一**：包含行程方向、金额等信息
5. **金额核对**：滴滴发票 + 通行费 ≈ 实际付款
