# 高校报销文件整理

本目录用于整理学术活动报销材料（讲座、论坛、会议、差旅、文印、图书等）。

**使用前加载 skill：**
```
skill_view(name='university-reimbursement')
```

---

## 整理原则

1. **一笔付款一个事项**，对应 Excel 一行
2. 同一笔费用的发票、付款记录、订单、行程单放入同一个文件夹
3. 文件夹命名让人一眼看懂：`日期_类型_说明_金额`（如 `2026-06-15_机票_北京至A市_660 元`）
4. 文件命名保持简洁：`01_发票.pdf`、`02_付款记录_微信支付.png`、`03_订单.png`
5. 打车文件夹内顺序：发票→付款记录→订单→行程单→高速费
6. 图片文件必须重命名（不要留 IMG_XXXX.png）
7. 不删除、不修改原始文件
8. 不确定的就放"待确认"，不猜

## 常见场景整理要点

### 跨市滴滴/打车（重要）

跨市行程（如A市↔B市）的高速费处理：

- **高速费归入打车费文件夹**，不单独建文件夹
- 通行费分两部分：
  - **去程**：载客行程的高速费
  - **返程**：司机空车返回的高速费（由乘客承担）
- **方向判断**：看行程单的"入口"→"出口"
  - `入口：城际北线-广东城东站，出口：城际南线-广东B市站` → A市机场→B市
- **金额核对**：滴滴发票 + 去程高速费 + 返程高速费 ≈ 实付金额
- 差额一般在 10 元以内（平台服务费或司机多收）

### 多张分段发票

同一笔费用可能拆成多张发票（如高速费分 4 段），全部归入同一事项文件夹，文件名标金额区分。

### 重复文件

按票据索引和按行程索引两份汇总单只留一份（留票据版）。

### 图片识别

- 优先用 Hermes 视觉能力看图（`vision_analyze`）
- 视觉不可用时用本地 OCR：`python3 ~/.hermes/skills/productivity/university-reimbursement/scripts/read_image.py <路径> --brief`

## 执行流程

1. **先问清楚**：学校全称、税号、项目号、特殊要求；材料在哪
2. **扫描文件**：列出所有文件，PDF 用 pdftotext，图片用视觉/OCR
3. **归类**：按金额、日期、收款方、上下文判断哪些属于同一笔费用
4. **用户确认**：当分组不明确时，先列出来让用户确认，不要自己猜
5. **整理文件**：建文件夹 → 复制 → 重命名 → 排序
6. **生成 Excel**：序号、事项、日期、说明、金额、材料清单、备注
7. **交付**：列出每个事项的状态和需要用户确认的问题

## 历史教训

1. **不要猜高速费方向**——必须看行程单的入口/出口站名判断，否则搞反去程返程
2. **不要跳过图片文件**——支付截图里有金额、日期、收款方，必须处理
3. **不要自己假设**——不确定文件归属时，先问用户，绕三圈不如问一句
4. **文件名用日常语言**——"机票"不是"专家差旅费"，"打车"不是"市内交通费"
5. **司机返程高速费也算**——跨市单司机空车返回的钱乘客出
