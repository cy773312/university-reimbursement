## 这项目解决什么问题

学术报销的痛点不在于财务规则，而在于**整理本身**：

- 一次出差产生十几份文件：发票、微信截图、行程单、高速费票、酒店水单
- 文件类型杂：PDF、PNG、JPG 混在一起，命名乱七八糟
- 逻辑关系散：哪张图对应哪笔钱，哪张发票对应哪段行程，全靠人脑硬记

**这不是人该花时间做的事。** 这是 AI 擅长的：批量扫描、按规则归类、逐项核验、输出结构化的结果。

项目把整理规则写成 AI 能执行的文档，跑一次就知道效果。

## 快速开始

```bash
# 1. 安装
bash install.sh

# 2. 在你的报销目录里放好原始文件，复制模板
cp template/AGENTS.md ~/报销/

# 3. 运行
cd ~/报销 && hermes -s university-reimbursement
```

## 一个例子

整理前：一个文件夹里 28 个文件，PDF 发票、截图、行程单混在一起，文件名是 IMG_2430.png、26468880111…pdf 这种。

```

│   ├── 02_付款记录_微信支付.png
│   └── 03_订单.png
├── 2026-05-28_打车_机场至酒店_370 元/
│   ├── 01_发票.pdf
│   ├── 02_付款记录_微信支付.png
│   ├── 03_订单.png
│   ├── 04_行程单.pdf
```

## 它能处理什么

| 材料类型 | 自动识别内容 | 归并到 |
|---------|------------|-------|
| PDF 电子发票 | 金额、税号、收款方 | 对应费用项 |
| 支付截图（PNG/JPG） | 金额、日期、商户 | 作为付款记录 |
| 网约车行程单 | 时间、里程、费用 | 打车费 |
| 高速费发票 | 金额、路线 | 对应打车行程 |
| 酒店水单 | 房费、入住日期 | 住宿费 |
| 图书订单 | 金额、书名 | 图书费 |

## 设计原则

1. **AI 做归类，人做决策**——AI 扫描文件、按规则分组、呈现结果；不确定的地方标记出来让你确认，不替你做主
2. **命名让人一眼看懂**——"机票"比"专家差旅费"好懂，"打车"比"市内交通费"直观
3. **金额必须能对上**——发票 + 高速费 ≈ 实付金额，对不上就标出来
4. **通用而非特例**——核心逻辑覆盖大多数场景，特殊规则按需补充
