## 🧠 它怎么做到的

### 文件归并大法

AI 靠几个线索把文件串起来：

```
金额一致  →  发票 202.01 + 付款记录 202.01  →  同一笔打车
日期接近  →  5月28日的机票 + 5月28日的打车  →  同一次出差
收款方一致 →  都是"滴滴出行"  →  同一行程
```

### 跨市打车高速费（最容易搞错的地方）

跨市行程（比如深圳↔珠海），高速费分两段：

```
🧑 乘客去程：深圳机场 → 珠海      高速费 84 元
🚗 司机返程：珠海 → 深圳（空车）  高速费 84 元  ← 这也算你的
```

AI 根据行程单上的**入口→出口**判断方向：

```
入口：深中侧接线-广东机场站沿江
出口：广珠东线-广东珠海站
→ 深圳→珠海方向
```

### 图片识别双保险

| 路径 | 依赖 | 适用场景 |
|------|------|---------|
| 🦅 视觉路径 | 大模型原生看图 | 有 vision 能力的模型 |
| 📠 OCR 路径 | Tesseract 本地识别 | 纯文本模型/离线环境 |

先试视觉，不行就切 OCR，反正不放过任何一张图。

## 🙈 血的教训

这个项目从一次灾难性的整理经历中诞生。AI 三番五次搞反高速费方向、跳过图片文件、不跟用户确认就自作主张。以下是我们（人类 + AI）一起总结的：

1. **不要猜高速费方向**——看入口/出口站名，靠猜必翻车
2. **不要跳过图片**——支付截图里有金额和日期
3. **不确定就问**——绕三圈不如问一句
4. **名字起清楚**——"机票"比"专家差旅费"好懂
