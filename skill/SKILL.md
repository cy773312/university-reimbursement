# 高校讲座/论坛报销文件整理 Skill

## 使用场景

当用户有高校讲座、论坛、学术会议相关的报销材料需要整理时，使用本 skill。

典型用户：大学生（研究生/博士生为主）、课题组秘书、科研助理。
典型场景：为受邀专家报销差旅住宿、报销讲座劳务费、报销论坛场地茶歇、报销参会交通等。

本 skill 与通用报销整理 skill 的区别：

- 费用类别以高校财务认可的支出类型为准。
- 文件类型增加了高校报销特有的材料（邀请函、审批表、签到表、新闻稿等）。
- 识别规则加入了中国发票、电子发票、支付截图的特点。
- 标记财务审核中常见的退回风险点。
- 支持按经费项目（课题号）归集。
- **跨市网约车通行费处理**——见 `references/cross-city-rideshare-toll.md`。

## 工作流程（首次运行时）

当用户说"帮我整理报销材料"时，按以下顺序执行：

### 第一步：初始询问

先问用户索取关键上下文——不要假设自己知道。至少问清楚：

- **学校全称**（用于检查发票抬头，如"某高校"不是"中大"）
- **学校税号**（统一社会信用代码，用于检查发票）
- **经费项目号/课题号**（如果有，后续填入表格；无则写"未知"）
- **材料位置**：文件在哪里？如果用户说"都在这个目录里"，直接扫描目录。
- **特殊要求**：学校财务处有没有特别规定（如单张发票金额上限、住宿标准等）？

如果用户说"先看看吧"或"你看着办"——跳过询问，先扫描目录，整理后把发现的问题列给用户确认。

### 第二步：扫描与识别

1. 列出 `原始文件/`（或用户指定目录）下的所有文件。
2. 逐个识别文件类型（发票、付款记录、邀请函、行程单等）。
3. 提取关键信息：日期、金额、收款方、发票号、付款方名称。
4. 对于 PDF 文件，使用 `pdftotext` 提取文字内容。
5. 对于图片文件（微信/支付宝截图、照片发票），尝试 OCR 提取文字。

**图片识别策略（双路径）：**

图片识别有两个并行路径，根据当前模型和能力自动选路：

1. **视觉路径**（大模型原生看图）：调用 `vision_analyze` 工具，模型直接读图。需要 `auxiliary.vision.provider` 配置了支持多模态的模型后端。配置方式：
   ```
   hermes config set model.default auto
   hermes config set model.provider <你的provider>
   ```
   设置 `model.default: auto` 后，Hermes 会自动根据任务类型选模型——看图时切视觉模型，文本时切效率模型。

2. **OCR 路径**（本地离线兜底）：使用 `scripts/read_image.py` 通过 Tesseract 提取文字。支持批量处理，可提取金额、日期、收款方等关键信息。

使用原则：先调 `vision_analyze` 尝试视觉；如果模型没有多模态能力或 `vision_analyze` 返回 "No LLM provider configured for task=vision"，fallback 到：
```
terminal("python3 scripts/read_image.py <图片路径> --brief")
```

跨 skill 参考：Hermes 内置的 `ocr-and-documents` skill 覆盖 PDF/文档提取，本 skill 的 `scripts/read_image.py` 补充了图片 OCR 能力。

**OCR 失败时的降级处理（重要）：**

如果系统没有 OCR 工具（tesseract 未安装、无 vision 工具），**不要阻塞整个流程**：

1. 先完成所有 PDF 文件的提取和整理（PDF 通常包含完整的发票/行程单信息）。
2. 将图片文件暂时归入 `待确认/信息不完整/`，在 Excel 中标注"缺付款记录"或"待确认"。
3. 给用户两个选项：
   - **选项 A**：安装 tesseract（`brew install tesseract tesseract-lang`），然后继续 OCR。
   - **选项 B**：用户手动告知图片内容（金额、时间、收款方），Agent 补录。
4. 如果用户确认 PDF 数据已覆盖主要费用，图片只是重复的付款截图，可以跳过 OCR 直接交付。

**典型场景：** 用户有 10+ 张微信/支付宝支付截图，但对应的电子发票已在 PDF 中。此时图片只是"付款记录"附件，不影响核心凭证完整性。可以先整理 PDF 数据，图片作为辅助材料后续补充。

### 第三步：用户确认分组——先汇报，后动手（最重要的流程改进）

**在动手归类文件之前，必须先做这一步。** 不要自行假设文件之间的归属关系。

1. **列出你的分析**：说明哪些文件你判断属于同一事项、依据是什么（金额一致？日期接近？收款方相同？）。
2. **重点标出不确定的部分**：例如「有 4 张高速费发票，不确定是去程还是返程」或「有两张付款截图，金额不同，不清楚对应哪笔行程」。
3. **等用户回复确认后再操作**。不要在不确定的情况下直接复制文件并建文件夹——改起来比问起来麻烦得多。

这一步是为了避免**自行假设导致多次返工**。尤其在以下情况必须交互：
- 多张高速费发票需判断去程/返程方向
- 多个行程的通行费混在一起
- 图片文件 OCR 结果不确定

### 第四步：归类判断

按"归并判断依据"部分的规则，判断哪些文件属于同一报销事项。

- 匹配到的 → 记入同一事项。
- 不确定的 → **先问用户，不要自己猜**。放入"待确认"是最后的选择，不是第一步。
- 同一发票号重复出现 → 标记"疑似重复"。

### 第五步：整理文件

1. 为每个报销事项建立文件夹（命名格式见下文）。
2. 将文件复制到对应文件夹，按规范重命名。
3. 无法归并的文件放入对应的"待确认"子目录。

### 第六步：生成 Excel

1. 确定哪些列在本次数据中出现（必选列一定有，可选列看数据）。
2. 动态构建表头。
3. 填入数据行。
4. 检查风险点并填写"风险提示"列（如果该列存在）。
5. 输出到 `报销整理表.xlsx`。

### 第七步：交付给用户

按"最终交付"格式回复，列出需要用户确认的事项。

## 核心原则

1. 一笔付款或一个报销事项对应 Excel 表格中的一行。
2. 同一笔费用相关的发票、付款记录、行程单、邀请函等文件应放在同一个文件夹中。
3. 文件夹名称应能清楚说明该笔费用。
4. 文件夹内文件按用途重命名。
5. 无法确定归属关系的文件放入"待确认"。
6. **不删除、不覆盖、不修改原始文件。**
7. **不确定的信息不要猜测**，统一写"未知"或"待确认"。
8. 最终必须生成 `报销整理表.xlsx`。

### 跨市打车/滴滴行程的特殊规则

**通行费归属：** 跨市滴滴行程的通行费应放入打车费文件夹中，与滴滴发票、付款记录放在一起。

**通行费构成：** 跨市行程的通行费通常包含两部分：
- **载客行程通行费**：乘客乘坐路段的通行费（有发票 + 行程单）
- **司机空车返程通行费**：司机送完乘客后空车返回的通行费（可能有汇总单或无发票）

**方向判断：** 仔细分析通行费行程单上的"入口"和"出口"信息：
- 入口 = 出发地，出口 = 目的地
- 例如："入口：城际北线 - 广东城东站，出口：城际南线 - 广东B市站" = A市机场→B市
- 不要假设，要根据实际行程信息判断

**金额核对：** 滴滴发票金额 + 通行费金额 ≈ 实际付款金额
- 差额通常在 10 元以内，可能是平台服务费
- 如果差额较大，检查是否遗漏了司机返程通行费

## 常见陷阱与注意事项

### 跨市网约车高速费判断错误

**问题：** 容易搞反高速费的去程/返程方向。跨市行程中，司机空车返回的高速费也由乘客承担。

**正确做法：** 仔细看通行费行程单的"入口"和"出口"信息：
- 入口 = 出发地，出口 = 目的地
- 例如：`入口：城际北线 - 广东城东站，出口：城际南线 - 广东B市站` = A市机场→B市
- 去程（载客）高速费 + 返程（空车）高速费 ≈ 实际通行费

**金额核对公式：**
```
滴滴发票金额 + 去程高速费 + 返程高速费 ≈ 实际付款金额
```

### 先判断、后行动（核心教训）

**问题：** 不做交互就自行假设文件分组，导致多次返工。

**正确做法：**
1. 归类之前先列出分析：`"根据金额和日期，这些文件应该分为 X 组，你看对吗？"`
2. 用户确认再动手
3. 宁可多问一句，不要绕三圈

### 图片文件遗漏

**问题：** 跳过图片文件（支付截图等），导致材料清单不完整。

**正确做法：**
- 有视觉能力 → 直接用 `vision_analyze` 读图
- 无视觉能力 → 用脚本 OCR：`python3 scripts/read_image.py <路径> --brief`
- 图片必须重命名（不要保留 IMG_XXXX.png）

### 遗漏司机空车返程通行费

**问题：** 跨市滴滴行程中，只计算了载客行程的通行费，遗漏了司机空车返程的通行费。

**正确做法：** 跨市行程的通行费通常包含两部分：
1. 载客行程通行费（乘客乘坐路段）
2. 司机空车返程通行费（司机送完乘客后空车返回）

如果滴滴发票金额 + 通行费金额 < 实际付款金额，检查是否遗漏了司机返程通行费。

### 文件命名不唯一

**问题：** 同一类型文件在不同报销事项文件夹中使用相同名称，只能通过文件夹区分。

**正确做法：** 文件名应包含行程、金额、日期等标识信息，确保跨文件夹唯一性。图片文件不要保留原始文件名（如 IMG_0010.png），应重命名为有意义的名称。

### 金额核对不精确

**问题：** 未仔细核对发票金额、通行费金额与付款金额是否一致。

**正确做法：** 
- 滴滴发票金额 + 通行费金额 ≈ 实际付款金额
- 差额通常在 10 元以内（平台服务费）
- 如果差额较大，检查是否遗漏了通行费或其他费用

## 高校报销常见费用类别

高校讲座/论坛报销主要涉及以下费用类型。请在识别文件时优先匹配这些类别。

**日常用语对照表：**

| 日常用语（文件夹/文件名用） | 正式类别（财务用） | 说明 |
|---------------------------|-------------------|------|
| 机票 | 专家差旅费 / 参会交通费 | 飞机票 |
| 打车 | 市内交通费 | 滴滴/网约车，含高速费 |
| 住宿 | 专家住宿费 | 酒店 |
| 高速费 | 通行费 | 归入打车文件夹 |
| 文印 | 材料打印/印刷费 | 打印、复印、装订 |
| 书籍 | 图书资料费 | 教材、学术书籍 |
| 文具 | 办公用品 | 笔、本子、文件夹等 |
| 快递 | 快递费/邮寄费 | 邮寄材料 |
| 茶歇 | 茶歇费 | 会议茶歇 |
| 劳务费 | 专家讲座费（劳务费/咨询费） | 需配套身份证+邀请函 |
| 场地 | 场地租赁费 | 含设备租赁 |
| 注册费 | 会议注册费 | 需配套会议通知 |
| 用餐 | 工作餐/会议用餐 | 餐费 |
| 其他 | 其他费用 | 上述未涵盖 |

**命名原则：** 文件夹和文件名**用日常用语**，不用正式类别名。
- ✓ `2026-06-15_机票_北京至A市_660 元`
- ✗ `2026-06-15_专家差旅费_660.00_CNY_海南航空`

## 高校报销文件类型

讲座/论坛报销可能涉及以下文件类型。编号规则对应如下。

### 高校报销文件类型

#### 核心凭证（必须关注）

```text
01_发票（增值税普通发票/电子发票）
02_付款记录（支付截图、转账记录、刷卡小票）
03_订单记录（平台订单详情、预订确认）
04_邀请函（专家邀请函、讲座通知）
05_审批材料（讲座审批表、事前审批单、预算表）
06_行程单（机票、火车票行程单）
07_身份信息（专家身份证复印件、银行卡号、护照）
08_证明材料（签到表、新闻稿、活动照片、会议议程）
09_通行费发票/行程单（高速费、过路费）
10_其他材料
```

**区分订单记录和付款记录：**
- **订单记录**：平台订单详情页截图，显示行程起点终点、时间、里程等（如滴滴行程详情、携程订单详情、机票订单页）
- **付款记录**：微信/支付宝支付凭证截图，显示支付金额、时间、收款方（如微信支付凭证、支付宝账单详情）

两者都应保留，分别编号存放。

### 发票类型识别

在中国高校报销场景中，常见的发票/票据类型：

```text
增值税电子普通发票（电子发票）——最常见的发票形式
增值税纸质普通发票
增值税专用发票——少数高校可抵扣
财政票据（行政事业单位资金往来结算票据）
中央非税收入统一票据
定额发票——出租车、餐饮
机打发票/卷式发票
个人所得税代扣代缴凭证——讲座费完税证明
```

### 付款记录类型

```text
微信支付截图
支付宝账单截图
银行转账截图
银行柜台汇款回单
POS 签购单/刷卡小票
公务卡账单
现金收据（收条）
```

## 报销事项文件夹命名

**命名哲学：让人一眼看懂是什么钱，不写多余的字段。** 不需要币种（中国高校都是人民币）、不需要完整公司名、不需要正式费用类别。

格式：

```text
日期_简单说明_金额
```

示例：

```text
2026-06-15_机票_北京至A市_660 元
2026-06-15_打车_A市机场至B市_175.00 元
2026-06-15_住宿_B市某酒店_418 元
2026-06-16_机票_A市至天津_1250 元
2026-06-16_打车_B市至A市机场_175.00 元
```

**费用说明用日常语言**，不用正式类别名：

| 日常用语 | 正式类别 | 使用场景 |
|---------|---------|---------|
| 机票 | 专家差旅费 | 飞机票 |
| 打车 | 市内交通费 | 滴滴/网约车 |
| 住宿 | 专家住宿费 | 酒店 |
| 高速费 | 通行费 | 高速通行费（归入打车文件夹） |

日期优先级：

1. 消费/付款日期。
2. 开票日期。
3. 行程日期（机票、火车票）。

金额优先级：

1. 付款金额。
2. 发票金额。
3. 收据金额。

无法确认的字段使用：

```text
未知日期
未知金额
待确认
```

## 文件夹内文件命名

**命名哲学：文件编号+类型+关键信息，不写多余字段。** 文件夹名称已经提供了上下文，文件名保持简洁即可。

格式：

```text
编号_类型_关键信息.扩展名
```

示例（打车文件夹内）：

```text
01_发票.pdf                         # 滴滴电子发票
02_付款记录_微信支付_175.00 元.png  # 付款凭证
03_订单_滴滴行程.png                # 订单详情
04_高速费_去程_84 元.pdf            # 去程通行费
06_高速费_返程_84 元.pdf            # 返程通行费
```

文件类型编号：

```text
01_发票
02_付款记录
03_订单记录
04_高速费/通行费
05_行程单
06_其他材料
```

注意：
- 图片文件（IMG_XXXX.png）必须重命名，不要保留原始文件名
- 同一文件夹内文件名天然是唯一的，不需要跨文件夹唯一
- 高速费文件标注"去程"或"返程"方向

## 归并判断依据

判断多个文件是否属于同一报销事项时，优先使用以下依据：

1. 金额一致或高度接近。
2. 币种一致（中国高校场景几乎都是 CNY，但有国际支出时需注意）。
3. 收款方/商户/人员名称一致或明显相关。
4. 日期接近。
5. 发票号一致（注意电子发票的发票号唯一性）。
6. 订单号、交易号、预订号一致。
7. 文件内容描述同一事件（如同一讲座、同一专家、同一行程）。

### 高校场景中的典型归并示例

可归并：

- 同一专家的邀请函 + 专家劳务费发票 + 劳务费转账记录 + 身份证复印件。
- 同一专家的机票行程单 + 电子发票 + 支付截图。
- 同一酒店的住宿费发票 + 支付截图。
- 同一讲座的茶歇费发票 + 支付截图 + 签到表。
- 同一场论坛的材料打印费发票 + 支付截图 + 打印清单。
- 同一会议的注册费发票 + 付款记录 + 会议议程。
- **跨市滴滴/网约车行程**：打车发票 + 付款记录 + 订单截图 + **往返通行费发票**（去程 + 司机返程空车）。

### 跨市网约车通行费归并规则（重要）

跨市行程（如A市机场↔B市）的通行费处理有特殊规则：

1. **通行费应归并到打车费文件夹中**，不是单独建文件夹。
2. **司机返程空车的高速费也应算入该行程**——跨市订单司机通常空车返回，返程高速费由乘客承担。
3. **通行费方向判断**：根据行程单上的入口/出口站名和时间判断方向，不要仅凭开票日期。
   - 去程：上车地点附近入口 → 下车地点附近出口
   - 司机返程：下车地点附近入口 → 上车地点附近出口（空车）
4. **金额核对**：打车发票金额 + 通行费金额 ≠ 付款金额是常见情况，差额可能为：
   - 平台服务费
   - 司机返程高速费（无发票）
   - 动态加价/夜间服务费
   - 在"问题说明"中记录差额及可能原因即可，不必强行匹配。

不可强行归并的情况：

- 同一专家但不同行程（分开多次出差）→ 分别建文件夹。
- 同一发票号出现在两个事项中 → 放入待确认/疑似重复。
- 金额和日期接近但收款方不同 → 不要强行归并，放入各自文件夹。

### 基本原则

**表格的列根据实际材料动态生成，不是固定模板。**

保持表格简洁：列越少越好，每个字段让人一眼看懂。去掉用户没提的字段。

## 匹配状态判断

### 已匹配

满足：

- 有发票。
- 有付款记录。
- 金额和收款方一致。
- 费用类型合理对应。
- （如涉及劳务费）身份信息和审批材料齐全。

### 部分匹配

适用于：

- 金额略有差异（可能由手续费、税费、汇率造成）。
- 收款方名称不完全一致但明显相关。
- 缺少辅助材料（如邀请函、签到表、新闻稿等非核心凭证）。
- 有发票和付款记录但缺少审批材料。

### 缺付款记录

有发票、邀请函、行程单等，但没有对应付款记录。

### 缺发票或收据

有付款记录，但没有对应发票。

### 缺审批材料

有发票和付款记录，但缺少以下一种或多种：

- 事前审批表/讲座审批表
- 邀请函
- 签到表
- 活动新闻稿/总结

适用于高校"先审批后支出"的报销规则。

### 待确认

适用于：

- 无法判断文件是否属于同一事项。
- 关键信息不清楚（看不清、读不了）。
- 有多个可能匹配项。

### 疑似重复

适用于：

- 同一发票号重复出现（电子发票常见问题）。
- 多个事项金额、日期、收款方高度相同。
- 同一文件被多次归类。

## 中国发票识别规则

在提取发票信息时，注意以下要点：

### 增值税电子普通发票（电子发票）

- 发票代码 + 发票号：唯一标识。同一发票号出现两次 = 疑似重复。
- 购买方名称：应该是学校全称（如"某高校"或"北京大学"）。如果不对，标记为"发票抬头非学校全称"。
- 购买方纳税人识别号（税号）：必须填写学校税号。如果缺失，标记为"发票缺税号"——这是财务处最常退回的原因之一。
- 销售方名称：填入"收款方"字段。
- 金额（不含税）和价税合计：优先使用价税合计（含税总金额）。
- 开票日期：填入"日期"。
- 货物或应税劳务名称：帮助判断"费用说明"。
- 备注：可能包含重要信息如银行账号、订单号。

### 定额发票

- 无金额信息 → 金额为"未知"。
- 通常有发票专用章 → 确认章上单位名称。
- 需要连号或累计后算总额。

### 财政票据

- 不是发票，是行政事业单位收据。
- 部分高校认可，部分不认可。标记为"财政票据"。

### 出租车/网约车发票

- 注意行程起止点和金额。
- 需要确认行程日期和事由（讲座/论坛相关）。

### 火车票和机票

- 火车票：纸质报销凭证（蓝色卡片）或电子客票。
- 机票：航空运输电子客票行程单（发票性质），不是登机牌。
- 两者都是报销凭证可以替代发票，但需要配套付款记录。

## 微信/支付宝支付截图识别规则

中国高校场景中大量使用微信/支付宝截图作为付款记录。

读取截图时注意：

- 收款方名称：可能与发票销售方不一致（第三方代收）。
- 实付金额：填入"金额"字段。
- 付款时间：填入"日期"。
- 交易单号：可用于查重。
- 支付方式：微信支付或支付宝。
- 订单备注：可能包含与讲座/论坛相关的说明。

如果截图上的收款方和发票销售方不一致，必须在"问题说明"或"风险提示"中标注。

## 高校财务退回风险点（优先检查）

在完成整理后，逐项检查以下常见退回原因。如果发现，在"风险提示"中标记：

1. **发票抬头错误**——发票上的购买方不是学校全称。
2. **发票缺税号**——购买方纳税人识别号未填写或填写错误。
3. **电子发票重复报销**——同一发票号出现在多个事项中。
4. **缺少事前审批**——有支出但没有找到审批材料。
5. **缺少邀请函**——讲座/论坛专家费没有对应的邀请函。
6. **缺少参会证明**——差旅费没有会议通知、议程或签到表。
7. **支出超标准**——住宿费/劳务费超过学校规定标准（如讲座费超过限额）。
8. **发票内容不符**——发票上的品名与实际支出不一致。
9. **收付款不一致**——付款记录上的收款方与发票销货方不符。
10. **缺少刷卡小票/转账记录**——大额支出（通常>1000 元）缺付款凭证。
11. **发票跨年**——发票日期和报销年度不一致。
12. **项目号错误**——经费项目号不属于该学生/课题组的权限范围。

### 常见整理错误（避坑指南）

1. **未取得用户确认就动手整理——这是最大的错误源**。不要自行假设文件归属关系，先摆分析让用户确认。宁可多问一句，不要绕三圈。
2. **文件名过于冗长**——用户想看的是"机票"、"住宿"、"打车"、"高速费"，不是"专家差旅费"、"市内交通费"、"通行费"。**
   - 文件夹名：`2026-06-15_机票_北京至A市_660 元` ✓，不是 `2026-06-15_专家差旅费_660.00_CNY_海南航空` ✗
   - 文件名：`01_发票.pdf` ✓（文件夹已提供上下文），不是 `01_发票_航空运输电子客票行程单_北京至A市_660 元.pdf` ✗
   - 原则：写的让人能看懂就行，不要写多余的字段。
3. **漏掉司机返程高速费**——跨市网约车行程，司机空车返回的高速费应由乘客承担，需归并到同一事项。
4. **图片文件未重命名**——`IMG_XXXX.png` 等原始文件名无意义，应重命名为包含金额、日期、类型的描述性名称。
5. **不同事项文件名重复**——只靠文件夹区分不够，文件名本身应在所有事项中唯一。
6. **发票与付款金额不一致时强行匹配**——差额可能是平台服务费、司机返程高速费等，在"问题说明"中记录即可，不必强行归并到其他事项。
7. **订单记录与付款记录混淆**——订单记录是平台订单详情（显示行程、里程），付款记录是支付凭证（显示实付金额、支付方式），两者都应保留。
8. **打车文件夹内文件排序混乱**——文件编号应遵循：发票 → 付款记录 → 订单 → 行程单 → 高速费（按去程→返程排列）。行程单应在高速费之前。
9. **同一笔通行费保留两份重复的汇总单**——当遇到按票据索引和按行程索引两份汇总单时，只保留按票据索引的汇总单，删除另一份。
10. **高速费方向判断凭感觉不凭证**——不要猜"A市出发就是返程"或"B市出发就是去程"，要看通行费行程单上的入口/出口站名，根据实际地理方位判断。如果不确定，问用户。

## 高校报销标准参考

不同高校标准不同。以下为常见范围（仅供参考，以各校财务处规定为准）：

```text
讲座费/劳务费：院士/知名专家 3000-5000元/场，教授 2000-3000元/场
住宿费标准：一般不超过 350-500元/人/天（分城市等级）
市内交通费：据实报销，通常不限交通工具
差旅补助：包干制 100-200元/人/天
工作餐/会议餐：标准通常 40-100元/人/餐
茶歇费：一般不超过 30-50元/人
材料打印费：据实报销
```

学生报销时，如果金额明显超出上述范围，在"风险提示"中标注"超出支出标准"。

注意：这些标准仅供参考，不要写入表格本身。只在风险提示中做标记。
